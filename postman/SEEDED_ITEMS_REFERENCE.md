## Validation Rules

### Valid Requests

✅ **Using itemId (NEW)**:

```json
{
  "sessionId": "abc-123",
  "itemId": 33
}
```

✅ **Using productId (LEGACY)**:

```json
{
  "sessionId": "abc-123",
  "productId": 1
}
```

### Invalid Requests

❌ **Missing both IDs**:

```json
{
  "sessionId": "abc-123"
}
```

Error: "Either productId or itemId must be provided"

❌ **Invalid itemId** (0, negative, or non-number):

```json
{
  "sessionId": "abc-123",
  "itemId": 0
}
```

Error: "itemId must be a positive number"

❌ **Missing sessionId**:

```json
{
  "itemId": 33
}
```

Error: "sessionId should not be empty"

## Troubleshooting

### "Item not found" Error

**Cause**: Using an item ID that doesn't exist or is inactive

**Solution**: Use IDs from 1-67 (all are active in seeded database)

### "productId must be a positive number" Error

**Cause**: Trying to use `itemId` but server expects `productId`, or validation decorator issue

**Solution**:

- Use `itemId` instead of `productId` for new model
- Ensure both are marked `@IsOptional()` in DTO
- Rebuild: `npm run build`
- Restart: `npm run dev`

### Empty Cart After Adding Items

**Cause**: Using different session IDs or cart not persisting

**Solution**:

- Use the same session ID from "Generate New Session" response
- Store session ID in Postman environment variable
- Check database: `SELECT * FROM cart WHERE session_id = 'YOUR_SESSION';`
